class SimControl¶

Bases: object

SimControl provides the basic interface to a CarMaker simulation process.

Its main tasks are: * Starting and stopping the simulation of a Variation * Checking simulation status and results * Synchronization of automation script and simulation process

Further capabilities are added by its subclasses.

async set_gpusensors(gpusensors)¶

Specify a list of GPUSensors and GPUSensorNX objects which may be used by the SimControl during simulation.

If a Variation is started by the Runtime, it is automatically taken care of this.

If GPU Sensors are supplied manually, the required Sensor Clusters of a Variation can be queried by a call to Variation.get_required_sensor_clusters

If a Test Run is expecting a mix of GPUSensor and GPUSensorNX objects due to Sensor Clusters with UseMovieNX enabled, the list must contain enough GPUSensorNX objects.

The order of the GPUSensor and GPUSensorNX objects can be arbitrary. SimControl will internally assign GPU SensorNX processes to suitable clusters.

Parameters:

gpusensors (list[Union[GPUSensor, GPUSensorNX]]) – A list of GPUSensor(NX) objects

Return type:

None

References

cmapi.GPUSensors, cmapi.GPUSensorNX
