class SimControlInteractive¶

Bases: cmapi.simcontrol.SimControl

An extension to SimControl using an ApoConnection to communicate with the simulation program during simulation.

Extends the basic capabilities by:

  • Pause, resume, abort simulation

  • Set simulation speed

  • Access to SessionLog “session log” during simulation

  • Extended synchronization SimCondition “conditions”

  • SimIO with * DVA * Remote log to session log

  • Quantity ApoSubscription “subscriptions”

  • Access to low level apo messages

async stop_master()¶

Stops the master application process by sending an APO signal.

After several unsuccessful attempts, the process is killed with operating system signals. The number of APO attempts can be configured with the config key “SimControlInteractive.ShutdownAttemptsN”

async start_and_connect()¶

Starts the simulation program and establishes an apo connection as a background task.

The simulation program must be a CarMaker object supplied separately by SimControlInteractive.set_master.

The same behavior can be achieved by separate calls to SimControlInteractive.start_master and SimControlInteractive.connect. If an ApoServer is used as master, e.g. only SimControlInteractive.connect should be used.

A simulation can be started by SimControlInteractive.start_sim, when this method has successfully returned.

Return type:

None

async disconnect()¶

Disconnect the SimControl from the ApoServer of the underlying simulation program.

Also disconnects all ApoSubscriptions and cancels the SessionLog background refresh. To allow file system synchronization before the final SessionLog update, a delay can be specified via two config keys “SimControlInteractive.DisconnectDelayErrorSec” and “SimControlInteractive.DisconnectDelayOkSec” which are used after an error and after successful simulation respectively.

create_rtexpr_condition(expression)¶

Creates and returns a “real time expression” condition.

The condition is registered with the currently connected master application via APO. The expression is evaluated every simulation cycle in the simulation program.

The condition can be awaited on client side. It should be noted, that there will be some latency introduced by the network transmission, after the expression has been triggered in the simulation program.

Parameters:

expression (str) – A string containing a valid CarMaker RTExpression

Returns:

An RTExprCondition object

Return type:

cmapi.condition.RTExprCondition

References

condition.RTExpression

Examples

‘await sim_control.create_rtexpr_condition(“first(Car.v>10)”).wait()’

create_quantity_condition(condition_func, *quantity_names)¶

Creates and returns a “quantity condition” associated with the master application via APO.

The “quantities” (ApoQuantity) used in the expression are automatically subscribed in a background process. Evaluation of the condition takes place on client side. A delay may be introduced by the network transmission of the values via APO.

The quantities are subscribed with a default update rate of 100Hz.

Parameters:

  • condition_func (Callable) – A callable. - Must evaluate to True to indicate that the condition has triggered. - Must accept one additional parameter of type ApoQuantity for each quantity name in quantity_names. These ApoQuantities correspond to the supplied quantity names and may be used in the expression.

  • quantity_names (list[str]) – A quantity name string or multiple quantity name strings in a list.

Returns:

A QuantityCondition object

Return type:

cmapi.condition.QuantityCondition

References

condition.QuantityCondition aposubscription.ApoQuantity

Example

‘sim_control.create_quantity_condition(lambda car_v: car_v > 10.0, “Car.v”)’

start_storage_save_all(ms_preceding=0)¶

Starts continuous saving of simulation data to file, as if StorageMode.save was specified.

May be followed by SimControlInteractive.stop_storage_save_all to end the capture and finalize the output file.

Parameters:

  • ms_preceding (int) – Milliseconds from the buffer to include at the beginning.

  • values (Special)

  • -1 – Add all samples in buffer

  • 0 – Don’t include buffered data (default)

Return type:

None

stop_storage_save_all()¶

Sends a message to the server to halt storing data.

Only if continuous saving of simulation data is active.

Return type:

None

clear_storage_buffer()¶

Discards all simulation data in the buffer.

Return type:

None

save_storage_buffer(ms_preceding=-1, ms_following=0)¶

Save simulation data of the timeframe defined by ms_preceding and ms_following into output file.

Parameters:

  • ms_preceding (int) – Milliseconds from the buffer to include at the beginning. Special values: -1: Add all samples in buffer (default) 0: Don’t include buffered data

  • ms_following (int) – Milliseconds from now to append at the end. Special values: -1: Add all following samples. This does not generate an output file until SimControlInteractive.stop_storage_save_all is called. 0: Don’t include any following samples. (default)

Return type:

None

async await_condition(condition)¶

Awaits the given SimCondition object belonging to this SimControl.

Normally this is equivalent to calling await condition.wait(). However, await_condition additionally allows to exit the waiting prematurely, if a simulation error occurred, while directly awaiting the SimCondition could have resulted in an infinite wait.

Parameters:

condition (cmapi.condition.SimCondition) – A SimCondition object

Raises:

SimulationError – Raised, if a simulation error occurred while waiting for the condition.

Return type:

None
