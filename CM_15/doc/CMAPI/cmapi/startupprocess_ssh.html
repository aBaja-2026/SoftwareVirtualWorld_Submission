StartupProcessSSH¶

class StartupProcessSSH¶

Bases: cmapi.startupprocess.StartupProcess

StartupProcess implementation which starts executable on a remote host via SSH.

Requires the SSH client and server to be properly set up (non-interactive sign in).

__init__()¶

Create StartupProcessSSH instance.

Return type:

None

async pipe_coroutine_function(process)¶

Coroutine function to handle the output of the process.

Returns immediately if no stdout handlers are set. Reads lines from the process’s stdout and calls the handlers with the output.

Parameters:

process (asyncio.subprocess.Process) – Process object to read output from.

Return type:

None

async start(host, exe_path, args, env_vars)¶

Start the Process on remote machine through SSH.

Uses the arguments:

-X: Enable X11 forwarding -o PubkeyAuthentication=yes: Use public key authentication -o UserKnownHostsFile=/dev/null: Ignore known hosts file -o StrictHostKeyChecking=no: Disable strict host key checking -n: Redirect stdin from /dev/null

Executes the given executable with the provided arguments and environment variables. Starts a coroutine to handle the process’s output and creates a PipeCoroProcessWrapper.

Parameters:

  • host (str) – Name of host as string.

  • exe_path (pathlib.Path) – Path to the executable to run on the remote host.

  • args (list) – List of arguments to pass to the executable.

  • env_vars (dict) – Dictionary of environment variables to set for the process.

Return type:

cmapi.startupprocess.PipeCoroProcessWrapper

class StartupProcessSSHApoclient¶

Bases: cmapi.startupprocess_ssh.StartupProcessSSH

StartupProcess implementation to start APO client programs on remote hosts.

APO clients which support startup option -apoprefix are started via SSH. Availability of APO services on the specified host is ensured by automatically starting apobroker daemon. Afterward, the started APO client can be identified via APO by its apoprefix.

__init__()¶

Create StartupProcessSSHApoclient instance.

Return type:

None

async start(host, exe_path, args, env_vars)¶

Start the APO client process on a remote host via SSH.

Parameters:

  • host (str) – Name of host as string.

  • exe_path (pathlib.Path) – Path to the executable to run on the remote host.

  • args (list) – List of arguments to pass to the executable.

  • env_vars (dict) – Dictionary of environment variables to set for the process.

Returns:

Wrapper for the started APO client process.

Return type:

PipeCoroProcessWrapper

Raises:

StartupError – If no matching APO client is found on the host within the timeout.
