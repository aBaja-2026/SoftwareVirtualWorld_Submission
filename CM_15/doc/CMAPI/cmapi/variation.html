class Variation(name='')¶

Bases: object

Variation corresponds to one simulation, which may or may not be already run.

Initially a Variation holds simulation input data in form of parametrizations, initial configuration of the simulation program, etc.

All simulation output data is stored in the Variation object and can be accessed after simulation.

Variation may be adapted by the user to hold additional input data or results. This data can be accessed e.g. from a customized VariationExecutionPolicy or directly from the top-level automation script.

A variation can be executed using a SimControl or Runtime object. For the latter, execution can be affected by the VariationExecutionPolicy owned by the variation.

There are several ExecutionStates of the variation and not all functions can be called from all execution states.

State transitions are: - Variation(): ExecutionState.created - Variation.create_tmp_testrun(): ExecutionState.configured - run Variation: ExecutionState.running - finish variation: ExecutionState.success, ExecutionState.error or ExecutionState.aborted - Variation.reset(): ExecutionState.created

get_resource(name)¶

Return a resource associated with this variation.

Before a resource can be accessed by this function, the resource requests of the variation must have been satisfied, e.g, supplying them with a Runtime.

Should not be called before ExecutionState.configured.

Parameters:

name (str) – Name string of the requested resource

Returns:

The value of the first resource matching the name or None, if no resource by this name is available.

Return type:

int | None

get_resource_descriptions(name='')¶

Returns all resources required by this variation.

Instead of accessing resource values via Variation.set_resource / Variation.get_resource, they may be accessed by their corresponding resource description objects, which contain information about the resource to be requested and, once supplied, a reference to the resource value itself.

Should not be called before ExecutionState.configured.

Parameters:

name (str) – Resource name string.

Returns:

List with ResourceDescription objects for resources used by this variation.

Return type:

list

create_tmp_testrun()¶

Finalize the variation payload.

A copy of the original Test Run is created: Variation.tmp_testrun with all referenced sub-parametrizations replaced by tmp files.

A second copy of the original Test Run is created: Variation.tmp_testrun_knval, with all named and key values substituted

Transitions into execution state ExecutionState.configured
