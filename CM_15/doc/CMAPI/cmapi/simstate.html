SimState¶

class ConditionSimState(value)¶

Bases: enum.Enum

Simulation states which can be awaited.

An ConditionSimState value can be passed to await_sim_state() functions to suspend the execution of the calling thread until the simulation reaches the given state. ConditionSimState is a simple version of ApplicationSimState, in which multiple values of ApplicationSimState are combined.

running: ApplicationSimState.running idle: ApplicationSimState.idle finished: ApplicationSimState.finished undefined: All remaining ApplicationSimState values

undefined = 1¶
idle = 2¶
running = 3¶
finished = 4¶
class ApplicationSimState(value)¶

Bases: enum.Enum

Class containing the internal simulation states of the CarMaker simulation process.

The values correspond to the ss2 message man_no field, with the constraint, that all values >= 0 are mapped to 0 which means ‘running’. Values > 0 are dynamically determined by the executed mini maneuver (dependent on the executed Test Run) and cannot be represented by the enum class.

class SimState¶

Bases: object

SimState reflects the execution state of a CarMaker simulation.

This includes - ConditionSimState (states which can be awaited with a SimStateCondition) - ApplicationSimState (internal simulation state) - Simulation completion state

Only available as part of SimControl, which undertakes the task of synchronizing the SimState object with the actual state of the simulation process. The actual extent of simulation state related information depends on the used SimControl implementation.
